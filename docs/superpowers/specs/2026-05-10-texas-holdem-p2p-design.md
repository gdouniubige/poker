# 德州扑克 P2P 本地对战 App — 设计说明

## 背景与目标

做一个仅限朋友之间当面玩的德州扑克（现金局），不涉及赌博、不涉及充值、不涉及陌生人匹配。全体用手机浏览器打开就能玩，不需要安装 App，不需要注册账号，不需要审核备案。

## 核心决定

| 维度 | 决定 |
|---|---|
| 玩法 | 德州扑克、现金局模式 |
| 人数 | 2–9 人 / 桌 |
| 连接方式 | P2P（房主手机开热点，其他人连热点 WiFi） |
| 进入方式 | 输入昵称、加入房间码 |
| 客户端 | 手机浏览器（PWA，可添加至桌面） |
| 界面 | 竖屏优先 |
| 筹码单位 | 分 |
| 盲注 | 固定（现金局不升盲） |
| 技术栈 | Vue 3 + Vite + PeerJS (WebRTC) + TypeScript |

## 架构总览

```
房主手机 (开热点)
   ├─ 运行游戏引擎（发牌、结算）
   ├─ WebRTC 星形中心节点
   └─ 8 条 P2P 通道连接各玩家
        ↓ (热点局域网)
   玩家1..8 手机 (浏览器)
   ├─ 接收game_state，渲染牌桌
   └─ 发送 action (fold/check/call/raise/all_in)
```

信令层：PeerJS 公共信令服务 (peerjs.com)，仅用于建立 WebRTC 连接（<1KB）。游戏数据全程走局域网 P2P 数据通道，不经过外网。

## 页面设计

### 首页

- 大标题 + 两个按钮：「创建房间」「加入房间」
- 创建房间 → 弹窗输昵称 → 生成 6 位房间码
- 加入房间 → 输房间码 + 昵称
- 首次使用时提示开启 PWA 安装

### 房间大厅

- 显示房间码（可复制、可分享）
- 玩家列表（头像占位 + 昵称 + 房主标签）
- 房主可见「开始游戏」按钮
- 满 2 人可开始

### 牌桌主界面（竖屏）

## 引擎设计

### 状态机

```
waiting → dealing → preflop → flop → turn → river → showdown → (回到 waiting)
```

每个阶段有一个下注圈（betting round）。下注圈结束条件：所有未弃牌玩家下注一致（或 All-in）。

### 手牌评估

- 输入：7 张牌（2 手牌 + 5 公共牌），对已弃牌玩家只用 5 张公共牌检查是否有资格摊牌
- 遍历 C(7,5) = 21 种组合，取最佳 5 张
- 牌型从高到低：皇家同花顺 > 同花顺 > 四条 > 葫芦 > 同花 > 顺子 > 三条 > 两对 > 一对 > 高牌
- 相同牌型按踢脚牌依次比较

### 底池

- 有人 All-in 且筹码不足跟注时产生边池
- 按玩家投入金额从小到大分层
- 结算时从最外层边池开始分配

### 现金局特性

- 盲注固定（如 1/2）
- 初始筹码可配置（默认 1000 分）
- 支持随时买入（补回到初始额度）
- 支持随时离场

## 通信协议

### 消息格式

房主 → 全体玩家：
```json
{"type":"game_state","data":{
  "phase":"flop",
  "community_cards":["Ah","Kh","Qd"],
  "players":[{"name":"小明","chips":250,"bet":20,"folded":false,"allIn":false},...],
  "pot":65,"current_player":"小红",
  "your_cards":["As","Ks"]
}}
```

玩家 → 房主：
```json
{"type":"action","data":{"action":"raise","amount":100}}
{"type":"join","data":{"name":"小明"}}
{"type":"rebuy","data":{}}
```

### 断线处理

- PeerJS 自动重连（10s 内）
- 重连后房主补发当前状态
- 房主断开 → 当前局打完，提示重开

## 界面状态

- 空闲：等待开始
- 游戏中（自己的回合）：操作栏可用，倒计时（可选）
- 游戏中（别人的回合）：等待，显示当前行动者
- 已弃牌：手牌变灰，观战
- 结算：弹窗展示赢家 + 牌型

## 不做的功能（YAGNI）

- 聊天系统（初期不做，当面聊）
- 计时器（初期不做，现场催）
- 统计数据（初期不做）
- 多桌（初期只做一桌）
- 房主转移（初期房主断就重开）
- AI 机器人
- 音效

## 验证方式

1. `npm run dev` → 手机浏览器打开 → 创建房间
2. 另一台手机扫码/输入房间码 → 加入
3. 重复验证 3 人局和 9 人局
4. 验证 All-in 边池正确性
5. 验证断线重连（切换 App 再回来）
6. `npm run build` → 部署到静态服务 → PWA 安装到桌面
